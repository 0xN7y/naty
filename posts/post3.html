What the Hell is a Kernel?

Alright, imagine your computer is like a massive empire. At the top of this empire sits the Kernel, like the freakin’ king. It manages all the resources cpu, memory, io device while making sure all the programs (your hello world app, processes, whatever) behave themselves.

It's the middleman between hardware and software. Your programs can't talk directly to the hardware, They gotta go through the kernel to get access. It handles the low-level tasks like talking to your CPU, memory management and file systems


When a program needs something, like reading a file or sending data over the network, it doesn’t just walk up to the CPU and demand it. Hell no. It has to ask the kernel first. This is done through system calls. Think of system calls as requests or commands that programs use to ask the kernel to do stuff.

How programs interact with the kernel

System Calls

When you run a program and it needs to do something the kernel controls, it uses a system call. like read(), write(), open(), close(), fork(), execve()..

User Space vs. Kernel Space

User space is where your programs run. It’s like a sandbox. Limited access, so you don’t screw up the system

Kernel space is where the kernel operates. This is the heart of the empire, yo. It has full access to everything, including memory and hardware

Interrupts

The kernel can get interrupted by hardware or software. Think of an interrupt as something that happens that requires immediate attention—like your CPU finishing a task or a program needing more memory


Kernel Code

So, what does the kernel code look like? It's some serious low-level stuff, yo. Kernel code is written in C and Assembly no JavaScript or Python here, bro. This is hardcore territory. Let's check out a simple example of a system call in the Linux kernel 

                
                    ssize_t sys_write(unsigned int fd, const char __user *buf, size_t count) {
                        // do some validation
                        if (!access_ok(VERIFY_READ, buf, count))
                            return -EFAULT;
                        
                        // Call the lower-level file system write operation
                        return vfs_write(fd, buf, count);
                    }

sys_write(): This is the system call for write(). When a program wants to write data to a file or the console, it calls this function


Why we love Kernel


Alright, now here's the juicy part why hackers are all up in the kernel's business. See, the kernel is the one piece of software that controls everything, so if you can mess with it, you can control the whole damn system. Here's why hackers have such a love affair with the kernel:

Rootkits

Hackers use rootkits to hide their activities in a compromised system. Rootkits are often kernel modules (code that can be added to the kernel) that modify how the kernel works. With a rootkit, a hacker can make it so that their malware is invisible to the operating system. You can hide files, network connections, and processes all thanks to kernel-level control.


Kernel Modules

Hackers often create kernel modules as part of their attacks. A kernel module is just a chunk of code that can be loaded into the kernel to extend its functionality think of it like adding new features to the kernel


That can be for hiding malware, modify system calls to change how the operating system behaves, hide connections...

The kernel is like the heart of the system, yo. It controls everything, from hardware to software, and if you can mess with it, you can do whatever the hell you want. Thats it for this part see you in a bit
